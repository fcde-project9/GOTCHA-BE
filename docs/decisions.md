# 설계 결정 사항

## 아키텍처 결정

| 항목 | 결정 | 이유 |
|------|------|------|
| social 테이블 | users에 통합 | 1인 1소셜 계정, 단순화 |
| location 테이블 | shops에 통합 (lat/lng) | 1:1 관계, 조회 성능 |
| 좌표 타입 | double | 카카오 API 호환, 충분한 정밀도 |
| 영업시간 | JSON (jsonb) | 요일별 관리 가능 |

---

## 비즈니스 결정

| 항목 | 결정 | 비고 |
|------|------|------|
| 가게 제보 | 즉시 등록 | MVP 단순화, 추후 승인제 검토 |
| 게스트 | 조회만 가능 | 찜/댓글은 로그인 필요 |
| 검색 | 지역 기반 | 가게명 검색은 Phase 2 |
| 정렬 | 거리순 기본 | |

---

## MVP 범위

### 포함
- 소셜 로그인 (카카오, 구글, 네이버, 애플)
- 지도 기반 가챠샵 조회
- 지역 검색 → 지도 이동 → 주변 가챠샵 표시
- 가게 상세 정보
- 찜 기능
- 가게 제보
- 댓글, 리뷰

### 제외 (Phase 2)
- 커뮤니티 (posts, post_comments)
- 채팅 (chats, chat_rooms)
- 문의하기 (inquiries)
